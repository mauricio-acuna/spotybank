## Palabras clave

Microservicios, Java, Spring Boot, Maven, arquitectura bancaria, modernizacion legacy, IA aplicada al desarrollo, DevSecOps, Kubernetes, OpenShift, observabilidad, performance, seguridad, educacion tecnica.

## Resumen ejecutivo

Esta obra usa Spotybank como caso de estudio para explicar como diagnosticar, anonimizar, documentar y modernizar un ecosistema de microservicios bancarios. El objetivo es mostrar un proceso repetible: partir de un codigo heredado con deuda tecnica, convertirlo en un entorno educativo seguro, detectar riesgos, proponer mejoras y secuenciar una migracion hacia practicas actuales de arquitectura, nube, seguridad, testing, observabilidad y performance. El enfoque permite usar el material como libro completo o como serie de fasciculos dirigidos a perfiles distintos de un equipo de software.

## Criterio editorial

- Mantener lenguaje tecnico claro, orientado a aprendizaje y toma de decisiones.
- Separar evidencia local, inferencias y decisiones pendientes.
- No incluir secretos, nombres de entidades reales, dominios corporativos ni autores previos.
- Convertir cada capitulo en una unidad aplicable: diagnostico, criterio, ejemplo Spotybank y ejercicios.
